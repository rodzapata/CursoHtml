    Etiquetas semanticas estructurales

    Estas etiqueta son las de marquetado y tienen un significado semantico para el navegador.
    llegaron para remplazar las Div que es una etiqueta contenedora pero que no tenian
    significado para el navegador, y que se volvia dificil de mantener.
    A partir del Html5 se crearon las etiquetas Header, naver, section, aside, footer, article, etiqueta main, define la seccion principal del documento, solo debe existir una seccion main por documento. etiqueta footer, respresenta el pie de pagina de un sitio web o una sección.
    eitqueta nav nos permite navegar, es donde irian los enlaces.
    etiqueta article, repreenta seccion autocontenido principal.
    etiqueta aside, representa una subseccion secundaria del contenido principal.
    se acostumbra a colocar la lista de contenido similar al contenido, o aviso publicitarios.
    etiqueta Section, representa una secion de contenido,
    etiqueta address, repreenta una direccion fisica o tambien informacion del contacto,
    por ejemplo Kra 5B No. 56C-93 Barrio valentina. o para contactar al autor ...

    • utilizando la etiqueta <Style> y </Style>
    • crear un archivo independiente Style.css donde se colocan los estilos a utilizar en la pagina

    Manejo de clases dentro del archivo Style.css

    las etiqueta debe tener Class="nombre" y en arhivo de Style.css, por ejemplo: en una lista ordernada quedaria de la siguiente forma: <Ol Class="nombre_clase" > y dentro del archivo de estilos debe iniciar con un punto, .lista-gatos{ color: brown; }
