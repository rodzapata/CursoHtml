    Etiquetas semanticas estructurales

    Estas etiqueta son las de marquetado y tienen un significado semantico para el navegador.
    llegaron para remplazar las Div que es una etiqueta contenedora pero que no tenian

    A partir del Html5 se crearon las etiquetas Header, naver, section, aside, footer, article, etiqueta main, define la seccion principal del documento, solo debe existir una seccion main por documento. etiqueta footer, respresenta el pie de pagina de un sitio web o una sección.
    eitqueta nav nos permite navegar, es donde irian los enlaces.
    etiqueta article, repreenta seccion autocontenido principal.
    etiqueta aside, representa una subseccion secundaria del contenido principal.
    se acostumbra a colocar la lista de contenido similar al contenido, o aviso publicitarios.
    etiqueta Section, representa una secion de contenido,
    etiqueta address, repreenta una direccion fisica o tambien informacion del contacto,
    por ejemplo Kra 5B No. 56C-93 Barrio valentina. o para contactar al autor ...

    Cajas de textos

    Estos no permite capturar texto por parte de los usuarios, se utiliza la etiqueta <input>
    se recomienda acompañarla con la etiqueta <label></label>, que contiene la
    descripcion de texto que le indica al usuario la informacion requerida.
    por ejemplo : 
          
        <form>
          <label>Nombre del estudiante: </label>
          <input> 
        </form>
    y nos muestra el siguiente resultado

    si queremos que la eliqueta label este asociada con la caja de texto, debemos colocarle a la
    caja de texto un Id, y a la etiqueta colocarle al atributo for el nombre del id de la caja de texto
    por ejemplo le colocaremos a la caja de texto nombEstudiante, quedando de la siguient manera:
    si el usuario le da un click a la etiueta el foco salta a la caja de texto al que está asociado.
    Es muy importante colocar la propiedad name que es el nombre del objeto, en este caso el nombre de
    la caja de texto nombEstudiante, es el nombre llave que es enviado cuando se envia el formulario y
    el valor digitado es el valor enviado.

          
        <form>
          <label for="nombEstudiante">Nombre del estudiante: </label>
          <input id="nombEstudiante"> name="nombEstudiante"
        </form>

    tambien existe la propiedad placeholder que es paracida a la etiqueta, pero en mensaje aparece dentro
    de la caja de texto. Ademas la propiedad type le indica a la caja de texto, como se va a comportar,
    puede ser text,button,checkbox, color, date, datetime, date-local, email, file, hiden, imagen,
    month, number,pasword, radio, submit, etc

    como puedes observar nos permite examinar las carpetas de nuestro dico para que escojamos cualquier archivo,
    pero si necesitamos escoger mas de uno tenemos que colocarle el atributo multiple quedando el codigo siguiente: 

          
        <input type="file" multiple>

    los tipos hiden son campos ocultos, el usuario no los ve, pero se utilizan mucho en programacion, cuando se necesita
    trabajar con una variable que tome un valor segun una condicion y sea enviada por nuestro formulario sin ser entrada directa del usuario.
